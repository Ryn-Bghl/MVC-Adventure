Chap. Flux d'une requête

Flux d'une requête — l'essentiel

Un utilisateur initie une requête (clic, formulaire, AJAX). Le Controller traite et valide, le Model gère les données, la View présente la réponse (HTML/JSON).

  • Initiation : navigateur → requête HTTP (méthode, URL).
  • Traitement : Controller (validation, authentification).
  • Données : Model (lecture/écriture, logique métier).
  • Rendu : View → réponse HTTP (code + contenu).

Exemple : Blog — Model = base d'articles, Controller = action «Créer», View = page affichant l'article.

Aide‑mémoire : Modèle = cerveau, Vue = écran, Contrôleur = chef d'orchestre.

MVC Adventure

Chapitre 3 – Cours : Le rôle du Controller

  • Le Réceptionniste de l'Application

    Le Contrôleur est le premier point de contact pour une action de l'utilisateur. Il reçoit la requête et sait exactement quoi faire. Par exemple, si un utilisateur clique sur "Sauvegarder", le contrôleur prend cette demande en charge.

  • Le Coordinateur des Tâches

    Le Contrôleur ne fait pas le travail lui-même. Il délègue. Il va dire au Modèle : "Prépare-toi à enregistrer ces informations". Ensuite, il dira à la Vue : "Affiche un message de succès à l'utilisateur".

  • Gestion des Entrées Utilisateur

    Le Contrôleur est spécialisé dans la gestion des entrées. Il récupère les données d'un formulaire, interprète les clics sur les liens et les boutons, et traduit ces actions en commandes pour le reste de l'application.

  • Le Cerveau Logique (sans les données)

    Il contient la logique de l'application : que faire quand l'utilisateur fait X ou Y ? Mais il ne contient ni la logique métier (comment les données sont structurées/manipulées, ça c'est le Modèle) ni la logique de présentation (comment ça s'affiche, ça c'est la Vue).
