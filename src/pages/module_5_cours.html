Chap5 Le rôle de la vue

Rôle de la vue

La vue affiche les données fournies par le modèle et permet aux utilisateurs d'interagir avec l'application.

  • Structure de la vue : Une vue est généralement un modèle HTML qui utilise le balisage Razor pour afficher des informations.
  • Séparation des responsabilités : Le modèle gère les données, la vue gère l'interface utilisateur, et le contrôleur gère la logique de l'application, ce qui rend le code plus lisible et facile à maintenir.

Exemple : Blog — Model = base d'articles, Controller = action «Créer», View = page affichant l'article.

Aide-mémoire : Modèle = cerveau, Vue = écran, Contrôleur = chef d'orchestre.
