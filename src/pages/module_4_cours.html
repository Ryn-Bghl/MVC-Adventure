Chapitre 4 – Cours : Le rôle du Modèle

    Le modèle (Model) dans l’architecture MVC est la partie qui gère les données, la logique métier et les règles de gestion de l'application.
    Il est responsable de récupérer, traiter, enregistrer ou modifier les données, généralement en lien avec une base de données.

    Il fonctionne indépendamment de l’interface utilisateur et du contrôleur, ce qui permet de réutiliser ou tester la logique métier sans dépendre de l'affichage.
    Lorsqu’une donnée est modifiée, le modèle peut notifier la vue pour qu’elle reflète les changements, assurant ainsi une bonne séparation des responsabilités dans l'application.
