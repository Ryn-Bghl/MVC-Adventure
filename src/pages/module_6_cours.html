Chapitre 6 – Cours : Flux Complet

  • Étape 1 : L'Action de l'Utilisateur

    Tout commence par une action de votre part. Par exemple, vous cliquez sur un lien "Afficher mon profil". Votre navigateur envoie une requête au serveur.

  • Étape 2 : Le Contrôleur Prend les Choses en Main

    La requête arrive au Contrôleur. C'est le premier à réagir. Il analyse la demande : "L'utilisateur veut voir son profil".

  • Étape 3 : Le Contrôleur Interroge le Modèle

    Le Contrôleur demande au Modèle : "J'ai besoin des informations du profil de l'utilisateur X". Il ne sait pas comment les obtenir, il fait juste la demande.

  • Étape 4 : Le Modèle Récupère les Données

    Le Modèle, qui est connecté à la base de données, exécute la requête pour trouver les informations de l'utilisateur X. Une fois trouvées, il les renvoie au Contrôleur.

  • Étape 5 : Le Contrôleur Prépare la Vue

    Avec les données en main, le Contrôleur choisit la Vue appropriée (par exemple, `profil.html`) et lui transmet les informations de l'utilisateur.

  • Étape 6 : La Vue Construit la Page

    La Vue reçoit les données et les insère dans le template HTML. Elle génère la page finale que vous allez voir, avec votre nom, votre email, etc.

  • Étape 7 : Affichage Final

    Le Contrôleur envoie la page HTML générée par la Vue à votre navigateur. Vous voyez enfin votre page de profil s'afficher ! La boucle est bouclée.
